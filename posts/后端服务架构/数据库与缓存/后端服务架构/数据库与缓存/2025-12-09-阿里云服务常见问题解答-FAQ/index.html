ECS (云服务器) 相关问题

Q1: ECS 实例无法远程连接怎么办？
A: 按以下步骤排查：

  1. 检查实例状态: 确保实例在控制台上处于“运行中”状态。
  2. 检查安全组规则: 确认实例所在安全组的入方向规则已放行了你的公网 IP 和对应的端口（Linux 默认 22，Windows 默认 3389）。
  3. 检查网络 ACL: 如果设置了网络 ACL，确保其规则也允许流量通过。
  4. 实例内部防火墙: 检查实例操作系统内部的防火墙（如 firewalld, ufw, Windows Defender 防火墙）是否放行了端口。
  5. 网络连通性: 使用 ping <实例公网IP> 和 telnet <实例公网IP> <端口> 测试连通性。

Q2: ECS 实例磁盘空间不足如何处理？
A:

  1. 清理无用文件:
  2. 扩容云盘:
    • 登录阿里云控制台，找到对应的 ECS 实例。
    • 在“云盘”部分，选择需要扩容的磁盘，点击“扩容”。
    • 扩容完成后，需要登录到 ECS 实例内部，执行文件系统的扩展操作（例如，使用 resize2fs 或 diskpart）。

Q3: 如何重置 ECS 实例的登录密码？
A:

  1. 登录 ECS 控制台。
  2. 找到目标实例，在“操作”列中选择“更多” -> “密码/密钥” -> “重置实例密码”。
  3. 根据提示完成验证并设置新密码，然后重启实例使新密码生效。

RDS (数据库) 相关问题

Q1: 应用程序无法连接到 RDS 实例？
A:

  1. 检查白名单: RDS 默认只允许指定的 IP 地址访问。请确保你的应用服务器的公网 IP 或所在的 ECS 内网 IP 已经添加到了 RDS 实例的 IP 白名单中。
  2. 检查内外网地址: 确认你的应用使用的是正确的数据库连接地址（内网地址或外网地址）。推荐在同一地域的 ECS 和 RDS 之间使用内网地址，以获得更低的延迟和更高的安全性。
  3. 账号与密码: 确认数据库的用户名和密码正确无误。
  4. 网络与端口: 确认安全组和本地防火墙已放行数据库端口（MySQL 默认 3306）。

Q2: 如何对 RDS 数据库进行性能优化？
A:

  1. SQL 优化:
    • 登录 DMS (数据管理服务) 或使用客户端连接到数据库。
    • 查看慢查询日志，找到执行效率低的 SQL 语句。
    • 使用 EXPLAIN 分析查询的执行计划，判断是否命中了索引。
    • 为 WHERE 和 ORDER BY 子句中频繁使用的列创建合适的索引。
  2. 参数调优: 在 RDS 控制台的“参数设置”中，根据你的业务负载调整核心参数，如 innodb_buffer_pool_size, max_connections 等。
  3. 升配或读写分离: 如果优化后性能仍不满足要求，可以考虑升级 RDS 实例的规格，或者购买只读实例，构建读写分离架构。

SLB (负载均衡) 相关问题

Q1: SLB 后端服务器健康检查失败，显示“异常”？
A:

  1. 检查健康检查配置: 在 SLB 控制台的监听配置中，确认健康检查的协议、端口、检查路径和 HTTP 状态码是否设置正确。例如，如果健康检查路径是 /health，请确保后端服务器能正常响应此路径的请求。
  2. 检查后端服务器:
    • 登录后端 ECS 实例，确认应用服务正在运行。
    • 检查实例的安全组和内部防火墙是否允许来自 SLB 服务器网段的流量（SLB 的网段通常是 100.64.0.0/10）。
    • 在后端服务器上，尝试用 curl http://127.0.0.1:<端口>/<健康检查路径> 命令，看是否能返回配置的正常状态码（如 200）。

Q2: 如何配置会话保持 (Session Stickiness)?
A:

  1. 在 SLB 控制台，找到监听器配置。
  2. 开启“会话保持”功能。
  3. Cookie 处理方式:
    • 植入 Cookie: SLB 会在 HTTP 响应中植入一个新 Cookie。客户端后续请求带上此 Cookie，SLB 就会将其转发到同一台后端服务器。你需要设置 Cookie 的超时时间。
    • 重写 Cookie: 如果你的后端应用本身就会生成 Cookie 用于会话，可以使用此模式。SLB 会重写你指定的 Cookie。

网络与安全组相关问题

Q1: 安全组规则的优先级是怎样的？
A: 安全组规则的优先级是数字越小，优先级越高。例如，优先级为 1 的规则会优先生效于优先级为 100 的规则。当有多条规则可能匹配同一个请求时，优先级最高的规则生效。

Q2: 如何区分安全组和网络 ACL？
A:

  • 安全组: 工作在实例级别，是有状态的（入方向规则允许后，对应的出方向流量会自动放行）。它相当于实例的虚拟防火墙。
  • 网络 ACL: 工作在子网级别，是无状态的（入方向和出方向的规则都需要单独配置）。它相当于子网的防火墙，可以为整个子网提供一层额外的安全保护。

OSS (对象存储) 相关问题

Q1: 如何实现防盗链？
A:

  1. 登录 OSS 控制台，进入你的 Bucket 管理页面。
  2. 选择“数据安全” -> “防盗链”。
  3. 设置 Referer 白名单。例如，填入 http://*.your_domain.com。
  4. 选择是否允许空 Referer。如果允许，则用户直接在浏览器地址栏输入图片 URL 仍然可以访问。
  5. 保存配置。
