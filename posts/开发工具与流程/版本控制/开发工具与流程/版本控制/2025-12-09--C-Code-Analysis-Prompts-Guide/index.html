1. 📖 概述

本文档提供了专门用于分析 C++ 项目（特别是 Drogon 框架）架构的详细提示词，帮助进行深度的源代码分析和依赖关系梳理。

🎯 分析目标

  • ✅ 完整的模块识别和分类
  • ✅ 详细的类关系分析
  • ✅ 依赖关系拓扑图
  • ✅ 架构特点总结

2. 🔍 核心执行流程

第一阶段：全面扫描与模块识别

  • 系统遍历 C++ 源文件（.h, .cpp, .cc, .hpp 等）
  • 分析项目目录结构，识别功能模块划分
  • 基于文件组织、命名空间和包含关系确定模块边界

第二阶段：模块依赖关系分析

  • 分析各模块间的头文件包含关系
  • 识别模块间的调用依赖和接口依赖
  • 建立清晰的模块依赖拓扑图

第三阶段：模块内完整类关系分析（关键重点）

对每个功能模块内的所有类进行深度分析：

  • ✅ 继承关系：完整的类继承链，包括多重继承和虚继承
  • ✅ 组合关系：类成员变量类型分析，识别聚合和组合
  • ✅ 依赖关系：方法参数、返回类型、局部变量类型依赖
  • ✅ 关联关系：友元关系、模板特化、类型别名关联
  • ✅ 完整性保证：确保不遗漏任何类，包括内部类、模板类、嵌套类

  • 未被归类到任何模块的类列表（如有）

5. ⚠️ 注意事项

5.1 完整性要求

  • 必须包含所有类：不因复杂度而跳过任何类
  • 特殊类处理：模板类、宏生成类需要特别标注
  • 解析困难：遇到困难时说明原因而非跳过

5.2 分析深度

  • 继承关系：完整的继承链，包括多重继承
  • 组合关系：详细分析成员变量类型
  • 依赖关系：方法参数、返回值、局部变量
  • 关联关系：友元、模板特化、类型别名

5.3 输出质量

  • 结构化输出：按照指定格式组织结果
  • 可视化图表：使用文本图表展示关系
  • 统计信息：提供完整的统计数据
  • 架构洞察：总结重要的架构特点
