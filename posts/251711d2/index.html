重要：安全须知

宝塔面板是一个拥有高权限的服务器管理工具，其安全性至关重要。请务必遵循以下基础安全实践：

  • 修改默认信息: 立即修改默认的 8888 端口、默认的用户名和密码。
  • 使用强密码: 为面板和数据库设置复杂的、无规律的密码。
  • 开启面板 SSL: 为你的面板访问地址开启 HTTPS，防止登录信息在传输过程中被窃取。
  • 定期更新: 及时将宝塔面板更新到最新版本，以获取最新的安全补丁。
  • 配置防火墙: 只开放必要的端口，关闭不使用的端口。
  • 定期备份: 养成定期备份网站文件和数据库的好习惯。

概述

宝塔面板（BT Panel）是一款简单易用的服务器运维管理面板，支持 Linux 和 Windows。它通过 Web 图形化界面，极大地简化了服务器的日常管理工作，如环境部署、网站创建、数据库管理、文件管理和安全监控等。

国际版: 宝塔面板有对应的国际版，名为 aaPanel。对于非中文用户，推荐使用 aaPanel，其官网为 aapanel.com。本指南以中文版的宝塔面板为例。

第一步：环境准备

1. 系统要求

  • 操作系统: CentOS 7.1+, Ubuntu 16.04+, Debian 9.0+
  • 内存: 建议 1GB 以上。
  • 磁盘空间: 建议 10GB 以上可用空间。
  • 要求: 一个纯净的、未安装过其他 Web 服务器软件（如 Nginx, Apache）的操作系统。

2. 开放端口

在安装前，请确保你的服务器提供商的安全组（或防火墙）已开放以下端口：

  • 8888: 宝塔面板默认端口
  • 80: HTTP
  • 443: HTTPS
  • 22: SSH
  • 21: FTP (如果需要)
  • 3306: MySQL (如果需要远程访问)

第二步：执行安装脚本

使用 root 或 sudo 权限登录你的服务器终端。

第三步：初始配置

  1. 登录面板: 使用上一步获取的地址、用户名和密码，在浏览器中打开并登录宝塔面板。
  2. 安装套件: 首次登录时，面板会推荐你安装一个 Web 环境套件，最常见的是 LNMP (Linux + Nginx + MySQL + PHP) 或 LAMP (Linux + Apache + MySQL + PHP)。根据你的需求选择一个并一键安装。
  3. 修改安全信息:
    • 进入左侧菜单的“面板设置”。
    • 修改面板端口: 将默认的 8888 修改为一个不常用的端口。
    • 修改面板用户和密码: 设置一个强密码。
    • 开启面板 SSL: 为你的面板域名（可以是 IP）申请一个 Let’s Encrypt 证书或粘贴你自己的证书，强制使用 HTTPS 访问。

核心功能使用

网站管理

  • 添加站点: 在“网站”菜单中，输入你的域名，选择 PHP 版本，创建数据库，即可快速创建一个网站。
  • 配置 SSL: 在站点的“设置”中，可以一键申请和部署 Let’s Encrypt 的免费 SSL 证书，并开启强制 HTTPS。

数据库管理

  • 创建数据库: 在“数据库”菜单中，可以快速创建数据库和用户，并管理权限。
  • 备份与恢复: 支持对数据库进行手动或定时的备份，并可以从备份中恢复。
  • phpMyAdmin: 面板内置了 phpMyAdmin，方便进行图形化的数据库管理。

文件管理

  • 提供一个功能强大的在线文件管理器，可以进行上传、下载、压缩、解压、在线编辑等操作。

监控

  • 在“监控”菜单中，可以直观地看到服务器的 CPU、内存、磁盘和网络负载的实时和历史图表。

常用命令

问题排查

Q1: 安装完成后无法访问面板地址?
A:

  1. 检查服务状态: 运行 sudo bt status 或 sudo systemctl status bt 确认面板服务是否在运行。
  2. 检查防火墙: 确认你的云服务器安全组和服务器内部的防火墙（如 ufw, firewalld）都已放行了你的面板端口。
  3. 检查端口占用: 使用 sudo netstat -tlnp | grep <你的端口号> 检查端口是否被其他程序占用。

Q2: 忘记了登录信息?
A: 登录服务器终端，运行 sudo bt default 即可查看当前的登录地址、用户名和密码。

Q3: 网站无法访问，提示 502 或其他错误?
A:

  1. 登录宝塔面板，检查“网站”->“设置”中的配置是否正确。
  2. 检查对应的 Web 服务器（Nginx/Apache）和 PHP 的服务状态是否正常。
  3. 在“文件”中查看对应网站的日志文件，分析具体的错误原因。
