Antigravity 登录认证完整解决方案

适配环境：Clash SOCKS5 7890 + Edge 浏览器 + 不需要 VS Code / CLI 兼容

第 1 部分：Clash 规则（必须）

你之前已经验证过：
👉 把 Antigravity OAuth 相关域名全部走”节点选择”（即代理节点）是可行的。

这是因为你的公司网络允许这些域名在代理链路上访问，没有额外拦截。

第 2 部分：Proxifier（必须）

你说得对，之前我确实漏写了 添加 Proxy Server 这一步，这次补上完整流程。

下面是你必须保证配置正确的 3 步：

✔ 第 2-1 步：添加代理服务器（第一步必须做）

Proxifier → Profile → Proxy Servers… → Add

填写：

  • Address: 127.0.0.1
  • Port: 7890
  • Protocol: SOCKS5
  • Authentication: NO

点 Check → 必须显示可连接
点 OK 保存

✔ 第 2-2 步：创建规则：Edge 直连（必须排第一）

Proxifier → Proxification Rules → Add

  • Rule name: Edge Direct
  • Applications: 选择 msedge.exe
  • Action: Direct

保存后：
👉 拖到最顶部（高优先级）

这是为了避免 Edge 被代理，避免 OAuth 出现循环、缩略 URL、回跳失败等问题。

✔ 第 2-3 步：创建规则：默认全部程序走 Clash（第二位）

Proxifier → Add Rule

  • Rule Name: Default via Clash
  • Applications: 留空（匹配所有应用）
  • Action: 选择你在第 2-1 步添加的 SOCKS5（127.0.0.1:7890）

保存

顺序如下：

  1. Edge Direct ← 不走代理
  2. Default via Clash ← 全部其他应用走代理（包含 Antigravity.exe）
